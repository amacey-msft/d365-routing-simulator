General Routing Settings

Configured under Routing in the Copilot Service admin center site map.

Preferred Representative Routing

Routes returning customers to their preferred rep (up to 3 per contact). Falls back to assignment logic or leaves unassigned if preferred rep is unavailable. Works with push-based workstreams only.

configure-preferred-agent

Skill-Based Routing

Matches work items to reps with matching skills and proficiency. Supports exact match and closest match algorithms. Skills can be set manually or via ML intelligent skill finder.

overview-skill-work-distribution

Shift-Based Routing

Routes conversations and cases based on rep shift schedules. Only reps currently on shift are eligible for assignment. Integrates with service scheduling calendars.

overview-unified-routing

Record Routing

Manage Unified Routing

Automates work distribution across channels (chat, voice, and cases). Enabled at the environment level. Once enabled, classification and route-to-queue rules process every work item.

overview-unified-routing

Setup Record Routing

Enables Dataverse record types (cases, emails, activities) for routing. Requires intake rules to determine which workstream handles each record. Supports Save & Route action in workspace.

set-up-record-routing

Intelligent Classification

Intelligent Skill Finder

AI Builder ML model that predicts required skills from work item text. Trained on historical data (or Excel import). Replaces manual classification rules. Requires AI Builder availability.

set-up-isf-model

Sentiment Prediction (Preview)

7-point sentiment scale (very positive → very negative). Uses Azure Cognitive Services. Sentiment score can be used in classification and route-to-queue rules for escalation routing.

use-sentiment-prediction-for-routing

Work Classification

Logical Classification Rules

Declarative if/then rules: "if IssueCategory = Billing, set Billing skill". Up to 10 rulesets per workstream, 100 rules each. Rules can reference 2 levels of related entity attributes.

configure-work-classification

ML-Based Skill Classification

Uses intelligent skill finder model. Sends work item text attributes to a published model. Returned tags are attached as skills. Supports iterative retraining from rep feedback.

ML skill classification

Capacity Profile-Based Rules

Appends capacity profiles to work items via classification. Assignment then checks reps have matching capacity profiles. Supports multiple profiles per workstream for tiered limits.

Capacity profile rules

Sentiment-Based Classification

Uses sentiment prediction output as a classification attribute. Route negative-sentiment items to specialized queues with experienced reps.

Sentiment classification

Queues & Capacity

Advanced Queues

Three types: Messaging, Record, Voice. Support queue prioritization (numeric), operating hours, overflow management, and fallback queues per workstream.

queues-omnichannel

Capacity Profiles

Define work item limits per rep with reset frequency (immediate or end-of-day). Supports custom per-rep limits, multiple profiles per workstream, and assignment blocking.

capacity-profiles
Assignment methods determine how work items in a queue are prioritized and assigned to eligible reps. Source: assignment-methods

Highest Capacity

Assigns to the rep with the most available capacity who matches skills and presence. Ties are broken by round robin. Works with both exact and closest skill matching.

Best for: Even workload distribution where rep bandwidth varies.

Advanced Round Robin

Assigns to matched reps in order of their last assignment time. Oldest assignment gets next item. Order is maintained per queue. Reps in multiple queues may get back-to-back from different queues.

Best for: Fair distribution regardless of capacity differences.

Least Active

Assigns to the rep with the fewest concurrent conversations. Uses "time since last capacity release" as tiebreaker. Default for voice and messaging queues. Not available for record queues.

Best for: Voice and messaging when you want reps to handle one thing at a time.

Custom Assignment

Build your own with prioritization rulesets (order work items) and assignment rulesets (match reps). Supports dynamic matching on related entity attributes and calendar schedule checks.

Key operators: Skills (exact/custom match), presence status, capacity, calendar schedule (is_working).

Assignment Cycle

The assignment cycle is triggered by: new work item arrival, rep presence change, capacity update, rep added to queue, or a periodic 5-minute timer (records). Each cycle picks top unassigned items and matches to eligible reps. Unmatched items retry in the next cycle.

Reps can decline up to 3 times (configurable 1–5). After max declines, auto-assignment skips that rep for that work item but they can still manually pick it.

Prioritization Rulesets

One prioritization ruleset per queue. Rules are ordered — first matching rule creates the top priority bucket. Within each bucket, items are sorted by the Order By attribute (FIFO, First Response By, etc.). Items not matching any rule go to the last bucket with FIFO ordering.

Example: Rule 1: Priority=High AND Category=Premium (ordered FIFO) → Rule 2: Category=Premium (FIFO) → Rule 3: Priority=High (by First Response By ASC)
